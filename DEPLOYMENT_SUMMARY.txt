┌────────────────────────────────────────────────────────────────────────┐
│  🚀 HOW TO DEPLOY (3 Simple Steps)                                     │
└────────────────────────────────────────────────────────────────────────┘

  STEP 1: Verify Setup
  ────────────────────
  $ ./check-vercel-setup.sh

  STEP 2: Go to Vercel Dashboard
  ───────────────────────────────
  👉 https://vercel.com/dashboard
  • Click "Add New Project"
  • Import your repository OR upload folder

  STEP 3: Configure & Deploy
  ───────────────────────────
  ⚠️  CRITICAL: Add ALL environment variables!
  
  In Vercel Dashboard → Project Settings → Environment Variables:
  
  Required (copy from .env):
  • NODE_ENV=production
  • MONGODB_URI=mongodb+srv://...
  • JWT_SECRET=...
  • JWT_REFRESH_SECRET=...
  • SUPER_ADMIN_EMAIL=...
  • SUPER_ADMIN_PASSWORD=...
  • SMTP_HOST=...
  • SMTP_EMAIL=...
  • SMTP_PASSWORD=...
  • IMAGEKIT_PUBLIC_KEY=...
  • IMAGEKIT_PRIVATE_KEY=...
  • IMAGEKIT_URL_ENDPOINT=...
  
  Then click "Deploy"!


┌────────────────────────────────────────────────────────────────────────┐
│  ⚠️  CRITICAL: MongoDB Atlas Configuration                             │
└────────────────────────────────────────────────────────────────────────┘

  MUST DO BEFORE DEPLOYING:
  
  1. Go to MongoDB Atlas Dashboard
  2. Click "Network Access"
  3. Click "Add IP Address"
  4. Select "Allow Access from Anywhere"
  5. Confirm with 0.0.0.0/0
  6. Click "Confirm"
  
  Why? Vercel uses dynamic IPs - this is required!

┌────────────────────────────────────────────────────────────────────────┐
│  🔍 TROUBLESHOOTING                                                    │
└────────────────────────────────────────────────────────────────────────┘

  Still Getting 500 Error?
  
  1. Check Vercel Function Logs (90% of issues show here!)
     → Vercel Dashboard → Project → Deployments
     → Click deployment → Functions tab → View Logs
  
  2. Verify Environment Variables
     → All variables must be in Vercel Dashboard
     → .env file is NOT deployed
  
  3. Check MongoDB Atlas Whitelist
     → Must allow 0.0.0.0/0
  
  4. Test /health First
     → If /health works but /api-docs fails, check logs
